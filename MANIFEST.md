# МАНИФЕСТ ПЛАТФОРМЫ
## Версия 1.0

### 1. Суть системы
Распределённая цифровая платформа для управления водными, береговыми и спортивно-туристическими активностями. Объединяет прокат, маршруты, соревнования, обучение и инклюзивные программы в единую экосистему, работающую онлайн и офлайн.

### 2. Цель
Создать инфраструктуру для:
- безопасной организации массовых и локальных активностей на воде
- управления прокатом через цифровые журналы
- проведения соревнований с автоматизированным судейством
- развития инклюзивных программ
- масштабирования через франчайзинг
- QR-доступа без установки приложений
- работы без интернета с офлайн-синхронизацией

### 3. Принципы
1. **Офлайн-первичность** — ключевые функции работают без интернета
2. **QR-доступность** — вход через QR-код, без установки приложений (PWA)
3. **Модульность** — независимые модули (прокат, маршруты, соревнования, обучение, инклюзия, франшизы)
4. **Событийность** — основная единица = СОБЫТИЕ (тренировка, аренда, матч, маршрут, программа)
5. **Географическая распределённость** — множество точек с локальными правилами и общими стандартами
6. **Человекоцентричность** — учёт уровней подготовки, ограничений и инклюзивных сценариев

### 4. Ключевая единица
**СОБЫТИЕ** — универсальный контейнер любой активности.
Все данные привязаны к событиям во времени и пространстве.

### 5. Три слоя архитектуры
| Слой | Описание |
|------|----------|
| **Сущности** (Data Layer) | Данные мира: пользователи, лодки, маршруты, события |
| **Связи** (Relationship Layer) | Как сущности связаны: кто участвует, где происходит |
| **Модули** (Functional Architecture) | Как система работает: прокат, спорт, гео, инклюзия |

### 6. Масштабирование
- Новые точки подключаются без изменения ядра
- Новые форматы событий — как модули
- Студенческие команды разрабатывают независимые части
- Франшизы работают автономно в рамках общих правил
