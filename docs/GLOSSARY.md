# СЛОВАРЬ СУЩНОСТЕЙ ПЛАТФОРМЫ
## ГОСТ-стиль, версия 1.0

---

## 1. ОБЩИЕ ПОЛОЖЕНИЯ

### 1.1 Назначение
Документ определяет единые термины, сущности и правила их трактовки.
**Обязателен для всех разработчиков, студентов и участников проекта.**

### 1.2 Принцип
Единицей системы является **СОБЫТИЕ**.
Все остальные сущности либо:
- участвуют в событии
- обслуживают событие
- являются инфраструктурой события

---

## 2. ОСНОВНЫЕ СУЩНОСТИ

### 2.1 ПОЛЬЗОВАТЕЛЬ
Физическое лицо, взаимодействующее с системой.
- **Роли:** участник, инструктор, судья, волонтёр, администратор точки, франчайзи
- **Участвует в:** событии, команде, аренде оборудования
- **Таблица:** `users`

### 2.2 СОБЫТИЕ
Базовая единица системы, описывающая любую активность.
- **Типы:** аренда, тренировка, маршрутный выход, матч, турнир, инклюзивная программа
- **Свойства:** время начала/окончания, место, участники, оборудование, результат
- **Связи:** маршрут, лодки, команды, GPS-трек, журнал, отчёт
- **Таблица:** `events`

### 2.3 ТОЧКА (БАЗА/ФРАНШИЗА)
Физическая локация проведения событий и хранения оборудования.
- **Типы:** базовая точка, франшизная точка, временная площадка
- **Функции:** выдача/приём оборудования, проведение событий, контроль безопасности
- **Таблица:** `points`

### 2.4 ФРАНШИЗА
Организационная единица, объединяющая точки.
- **Функции:** управление точками, стандартизация, отчётность, масштабирование
- **Таблица:** `franchises`

### 2.5 МАРШРУТ
Заранее определённый путь движения участников.
- **Состоит из:** контрольных точек (GPS-координаты)
- **Параметры:** длина, сложность, тип среды, ограничения
- **Таблица:** `routes`

### 2.6 КОНТРОЛЬНАЯ ТОЧКА
Фиксированная географическая точка маршрута.
- **Функции:** навигация, проверка прохождения, фиксация времени
- **Таблица:** `route_points`

### 2.7 ЛОДКА
Основной объект проката и спортивной активности.
- **Свойства:** тип, состояние, номер, цвет, грузоподъёмность
- **Связи:** хранится в точке, арендуется в событии
- **Таблица:** `boats`

### 2.8 КОМАНДА
Группа пользователей, объединённая в рамках события.
- **Состав:** участники (пользователи)
- **Участвует в:** матчах, турнирах, тренировках
- **Таблица:** `teams`

### 2.9 МАТЧ
Структурированное спортивное событие.
- **Содержит:** команды, судью, результат, время
- **Таблица:** `matches`

### 2.10 ТУРНИР
Система матчей с турнирной логикой.
- **Содержит:** сетку матчей, команды, правила подсчёта
- **Таблица:** `tournaments`

### 2.11 ИНСТРУКТОР
Ответственный за проведение и безопасность события.
- **Функции:** ведение тренировки, контроль участников, безопасность, обучение

### 2.12 СУДЬЯ
Лицо, фиксирующее результат спортивного события.
- **Функции:** фиксация счёта, подтверждение результата, контроль правил

### 2.13 GPS-ТРЕК
Последовательность координат движения пользователя или группы.
- **Формируется:** во время события
- **Используется для:** анализа маршрута, безопасности, отчётности
- **Таблицы:** `gps_tracks`, `gps_track_points`

### 2.14 ЖУРНАЛ
Регистрационная запись события.
- **Содержит:** кто участвовал, что использовалось, время, результат
- **Таблица:** `logs`

### 2.15 ОТЧЁТ
Агрегированный документ по результатам события.
- **Используется для:** франшизы, аналитики, контроля качества

### 2.16 АРЕНДА
Связь пользователя и лодки в рамках события-аренды.
- **Таблица:** `rentals`

### 2.17 ПЛАТЁЖ
Финансовая операция, привязанная к событию.
- **Таблица:** `payments`

## 3. ПРАВИЛА СИСТЕМЫ

| Правило | Описание |
|---------|----------|
| **Центральное событие** | Все процессы выражаются через СОБЫТИЕ |
| **Трассируемость** | Каждое событие оставляет журнал, GPS-трек, результат |
| **Распределённость** | Точки работают автономно, синхронизируются с системой |
| **Модульность** | Сущности не зависят от интерфейса |
| **Единый словарь** | Запрещено переименование без согласования |
