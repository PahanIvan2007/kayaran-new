## Обязательны для всех команд. Версия 1.0

## 1. ОБЩИЕ ПРАВИЛА

### 1.1 Единый формат ID
- Все ID — строки длиной 7 символов
- Формат: 1 буква префикса + 6 hex-цифр
- Префиксы: `U`(user), `E`(event), `P`(point), `F`(franchise), `R`(route), `B`(boat), `T`(team), `M`(tournament), `S`(match), `A`(rental), `G`(gps), `Y`(payment), `L`(log), `I`(inventory), `Z`(zone)
- Генерация: `generate_short_id(prefix)` на основе MD5 от UUID

### 1.2 Время
- Все даты и время: **UTC ISO 8601**
- Формат: `2026-05-08T14:30:00Z`
- В БД: `TIMESTAMPTZ`
- Преобразование в локальное время — на стороне клиента/API

### 1.3 География
- Все координаты: **WGS84 (EPSG:4326)**
- Формат: `lat DECIMAL(10,7), lng DECIMAL(10,7)`
- Для PostGIS использовать `GEOGRAPHY(Point, 4326)`

## 2. ЗАПРЕЩЕНО

| № | Правило | Почему |
|---|---------|--------|
| 1 | Создавать свои ID-форматы | Конфликты при синхронизации |
| 2 | Менять ENUM без согласования | Ломает другие модули |
| 3 | Переименовывать сущности | Разрыв контракта данных |
| 4 | Хранить GPS в JSON внутри событий | Убивает производительность |
| 5 | Дублировать users в модулях | Рассинхронизация данных |
| 6 | Удалять записи физически | Потеря истории |
| 7 | Хранить JSON там, где нужна нормализация | Нельзя делать запросы |

## 5. РАБОТА С ENUM

- ENUM'ы НЕЛЬЗЯ менять в существующих таблицах
- При необходимости нового значения: `ALTER TYPE ... ADD VALUE`
- Согласование: #arch в чате или issue

**Текущие ENUM:**
| Поле | Значения |
|------|----------|
| user.role | participant, instructor, judge, volunteer, franchise_admin, system_admin |
| user.status | active, blocked, deleted, pending |
| event.event_type | rental, training, route, match, tournament, inclusive_program, festival, corporate |
| event.status | planned, active, completed, cancelled, emergency |
| point.type | base, franchise, temporary_event, storage |
| boat.status | available, rented, repair, inactive, lost, reserved |
| route.difficulty | easy, medium, hard, adaptive |

## 6. КОНТРОЛЬ ВЕРСИЙ

- Схема БД: `db/` — файлы нумеруются: `001_core.sql`, `002_geo.sql`, ...
- API: `api/openapi.yaml` — автоспецификация
- Документация: `docs/` — Markdown
- Код модулей: `modules/{module_name}/`

---

## 7. ОФЛАЙН-ПРАВИЛА

1. **Все ключевые операции** должны быть доступны без интернета
2. **Локальное хранилище** — IndexedDB (PWA) или SQLite (native)
3. **Синхронизация** — при появлении соединения, фоновая
4. **Конфликты** — разрешаются по принципу "последняя запись побеждает" (LWW)
5. **Критичные данные** (SOS, экстренные) — отправляются при первой возможности с приоритетом

---

## 8. БЕЗОПАСНОСТЬ

1. **API** — только через HTTPS
2. **Авторизация** — JWT-токены
3. **Роли** — проверка на уровне API
4. **QR-коды** — содержат только ID, не персональные данные
5. **Персональные данные** — обработка согласно 152-ФЗ
