# Команда 2: Маршруты и GPS

## Домен
Геоданные, маршруты, контрольные точки, GPS-трекинг, офлайн-карты, зоны безопасности.

## Таблицы-владельцы
- `routes` — маршруты
- `route_points` — контрольные точки
- `gps_tracks` — сессии трекинга
- `gps_track_points` — точки треков (BIG DATA)
- `activity_zones` — зоны активности

## API
- `GET/POST /routes` — CRUD маршрутов
- `POST /routes/{id}/points` — точки маршрута
- `POST /gps/tracks` — старт трекинга
- `POST /gps/tracks/{id}/points` — загрузка точек
- `GET /gps/tracks/{id}` — получение трека
- `GET /zones` — зоны

## Задачи
- [ ] CRUD маршрутов с контрольными точками
- [ ] Пакетная загрузка GPS-точек (100+ точек за запрос)
- [ ] Получение трека в формате GeoJSON
- [ ] Зоны безопасности (опасные/безопасные)
- [ ] Офлайн-кеширование карт

## Особенности
- `gps_track_points` — таблица-гигант, обязательное партиционирование
- Использовать PostGIS (ST_Distance, ST_Within)
- Офлайн-режим критичен

## Связанные модули
- Core (events — маршрут привязан к событию)
- Rental (GPS для аренды)
- Inclusive (адаптивные маршруты)
