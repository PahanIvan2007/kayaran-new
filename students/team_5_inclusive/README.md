# Команда 5: Инклюзивность

## Домен
Адаптивные программы, профили ограничений, мед. данные, сопровождение, безопасность.

**Research-level домен.** Уникальный кейс платформы.

## Таблицы-владельцы
- `accessibility_profiles` — инклюзивные профили
- `medical_restrictions` — медицинские ограничения
- `adaptive_route_access` — адаптивные маршруты
- `escort_assignments` — назначение сопровождения

## API
- `GET/PUT /profiles/accessibility/{userId}` — профиль
- `POST /profiles/medical/{userId}` — ограничения
- `GET /routes/adaptive` — адаптивные маршруты
- `POST /escort` — сопровождение
- `GET /events/inclusive` — инклюзивные события

## Задачи

### База
- [ ] Профиль инклюзивности (тип ограничения, уровень мобильности)
- [ ] Медицинские ограничения (CRUD)
- [ ] Связь профиля с пользователем

### Исследование
- [ ] Классификация типов ограничений (зрение, слух, опорно-двигательный, когнитивный)
- [ ] Алгоритм подбора маршрута: нагрузка × тип ограничения × погода
- [ ] Система грейдирования участников (начальный → продвинутый)
- [ ] Тревожные кнопки и мониторинг состояния

### Развитие
- [ ] Адаптивные контрольные точки (доступность для каждого типа)
- [ ] Автоматическое назначение волонтёра-сопровождающего
- [ ] Отчёт по безопасности инклюзивного события

## Связанные модули
- Core (events, users)
- Geo (routes — адаптивные маршруты)
- Все модули (инклюзивность — сквозной слой)
